wrkdat displays as "999999" while the others display as normal dates.


This happens if oCol:cEditPicture of this column is "999999". XBrowse builds the picture from that column's cDataType, nDataLen and nDataDec. In turn, XBrowse takes this informatin from ::aStruct. It appears that XBrowse is reading the structure of the field 'wrkdat' from ::oorders as { "WRKDAT", "N", 6, 0 }. Surprising.

</s>oLbxo:oCol( 10 ):cEditPicture := nil<e>

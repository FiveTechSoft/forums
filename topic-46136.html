Actúa como un Arquitecto de Software Senior especializado en Fintech y Sistemas de Alta Frecuencia (HFT).

Tu objetivo es diseñar la arquitectura backend y el núcleo lógico para una nueva Casa de Bolsa (Stock Exchange/Brokerage). El sistema debe ser capaz de manejar miles de transacciones por segundo con latencia mínima.

Requisitos Técnicos:

Lenguaje: Prefiero Go (Golang) o Rust por su rendimiento y seguridad de memoria (elige el que consideres mejor para el Matching Engine).

Base de Datos: PostgreSQL para persistencia (ACID estricto) y Redis para caché de alta velocidad.

Arquitectura: Microservicios basados en eventos (Event-Driven Architecture) usando Kafka o RabbitMQ.

Entregables:

Diseño de Alto Nivel: Describe los componentes clave (API Gateway, Order Matching Engine, Wallet Service, Market Data Feeds) y cómo se comunican.

Modelo de Datos (Schema): Diseña las tablas SQL críticas para Users, Wallets, Orders (Limit/Market) y Transactions. Nota: Debes explicar cómo evitarás el doble gasto y números negativos en los balances.

Lógica del Core (Código): Escribe un ejemplo de implementación simplificada del Order Matching Engine (Motor de Emparejamiento) usando el algoritmo FIFO (Price-Time Priority). Debe mostrar cómo se cruza una orden de compra con una de venta.

Manejo de Concurrencia: Explica cómo manejarás las condiciones de carrera (race conditions) cuando dos usuarios intentan comprar la misma acción al mismo tiempo.

Por favor, prioriza la seguridad, la consistencia de los datos y la claridad en el código.

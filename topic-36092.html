Riassumendo, io ho inteso così:
1- Preparazione File XML:
Fatture PA - Deve sempre essere presente Cd.Ufficio (6 Caratteri) = Cd.Destinatario
B2B - Inserire il Cd.Destinatario (7 Caratteri) OPPURE Indirizzo PEC (se non presente Cd.Destinatario mettere '0000000')
Comunque deve essere sempre presente almeno 1 delle 2 informazioni: L'AE si occupa anche dell'invio del file al destinatario
2- Controllo File:
L'unico sito ufficiale che ho trovato è quello dell'AE:
il problema è che non si può mettere in automatico il nome del file (mi sono arrangiato con un Copia-Incolla), ne baipassare il CAPTCHA
3- Firma Digitale:
Fatture PA - Obbligatorio (Qualcuno sa come poterlo firmare con FW ?)
B2B - Non obbligatorio (XAdES / CAdES)
4- Spedizione: (Usando canale PEC il cliente non deve registrarsi: l'AE apre un canale per il cliente informandolo della PEC a lui dedicata)
canale PEC: la prima volta all'indirizzo sdi01@pec.fatturapa.it
in risposta l'AE invia l'indirizzo di riferimento a cui spedire i successivi files
5- Con il canale PEC non si possono gestire in automatico le risposte

Salve a tutti,

una cosa importante è anche quella dell'archiviazione sostitutiva,
non è sufficiente inviare il file XML via PEC ma le fatture vanno poi archiviate.

Per ditte di dimensioni medio/piccole una soluzione potrebbe essere questa:
se una ditta si registra a Fisconline l'agenzia delle entrate offre un servizio
gratuito di archiviazione di 15 anni. In pratica tutte le fatture emesse e ricevute
che passano dal SDI vengono archiviate automaticamente.

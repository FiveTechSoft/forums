una consulta:

700 valor a pagar
16% es iva?
4% otro impuesto?

si es así seria de la siguiente forma

700 / 1.16 = 603.45 lo que el 16% es 700 - 603.45 = 96.55
603.45 / 1.04 = 580.24 lo que el 4% es 603.45 - 580.24 = 23.21

700 - 96.55 - 23.21 = 580.24

entonces

580.24 = NETO
23.21 = 4%
96.55 = 16%

700.00 = total

todo depende de donde vas a calcular los % y el orden en que van, espero que te sirva

Mira se Ayuda:

Code (fw): Select all Collapse
// \SAMPLES\ARMIMPOS.PRG

#Include "FiveWin.ch"

FUNCTION ARM_IMPOS()

   LOCAL nValVentas, nValorTot
   LOCAL nValor83,   nValor16,   nValor04
   LOCAL nValor83_2, nValor16_2, nValor04_2
   LOCAL nAliqPorce, nAliq16Porc, nAliq04Porc

   // SET EXACT ON

   nValor83    := 0.00  // 83%
   nValor83_2  := 0.00  // 83%
   nValor16    := 0.00  // 16%
   nValor16_2  := 0.00  // 16%
   nValor04    := 0.00  // 4%
   nValor04_2  := 0.00  // 4%
   nValorTot   := 0.00  // total general

   nValVentas  := 700.00       // = Valor de Ventas
   nAliqPorce  := 0.83334857   // = 583.34  83.334857% aplicada sobre ventas.
   nAliq16Porc := 0.159992457  // = 583.34 x 16% = 93.3344
   nAliq04Porc := 0.0399938286 // = 583.34 x  4% = 23.3336

   // Criterio de Armando:

   nValor83    := nValVentas * nAliqPorce          // 583.34 // GET
   nValor83_2  := ( nValVentas * 83.334857 ) / 100 // 583.34 // FIXO

   ? nValor83, nValor83_2

   nValor16    := nValor83 * nAliq16Porc   // GET
   nValor16_2  := ( nValor83 * 16 ) / 100  // FIXO

   ? nValor16, nValor16_2

   nValor04    := nValor83 * nAliq04Porc    // 23.33   // GET
   nValor04_2  := ( nValor83 * 4 ) / 100    // 23.33   // FIXO

   ? nValor04, nValor04_2

   nValorTot    := ( nValor83 + nValor16 + nValor04 )

   ?  nValorTot

   nValorTot    := ( nValor83_2 + nValor16_2 + nValor04_2 )

   ?  nValorTot

RETURN NIL

// FIN / END

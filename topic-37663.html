FWmariadb library connects to the server with some initial settings including "sql_mode = 'PAD_CHAR_TO_FULL_LENGTH'"

We are unable to say if this is an issue with your version of MySql (5.2) or any settings in your my.ini.

At our end, we test the library with versions 5.3, 5.6, 8.0.15 and MariaDb. We never expreienced this problem. None of the other users who have been using did not report this problem so far. May be their versions are not as old as 5.2.

However, we are making these changes in the next version of FWH.

1) The library will first try to connect with this setting and on failure, will connect without this setting. So the connection will always succeed even on versions that do not support this setting.
2) Optionally, the programmer can use the new function:
FWMARIA_SET_PAD_CHAR_TO_FULL_LENGTH( .F. )
to prevent the library from trying this setting.

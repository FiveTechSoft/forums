-------------------------------------- NEW VERSION 10/11/2017 -------------------------

https://bitbucket.org/fivetech/fivewin- ... IVEDIT.EXE

News:

Now you can open one file that does not belong to the project and compile it
In this case, if you have a file that does not belong to the project, in order to compile the project you must open the first file of the project and be located in it
At the moment it is the easiest solution that I have found, although I have in mind other possibilities that can be implemented

Ahora puedes abrir un fichero que no pertenezca a un proyecto que esté abierto y compilarlo
En este caso, para que FE pueda distinguir entre compilar el fichero independiente o el proyecto abierto, ha de abrir el fichero inicial del proyecto y estar situado en él
Por el momento, es la solución más sencilla que he implementado, para no tener que cerrar el proyecto para compilar un fichero independiente del proyecto, pero más adelante lo implementaré de otra forma

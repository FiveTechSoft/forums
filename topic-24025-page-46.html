How to make a shortcut

[ SPANISH ]

Puede crear un acceso directo al fichero .EXE ( Fivedit.EXE ) como realiza normalmente, pero el programa acepta dos parámetros:

1.- Programa fuente ( .PRG ) para abrir directamente
2.- Fichero de configuración ( .INI ), si no se indica ninguno, el fichero de configuración tiene el nombre del .EXE,
en este caso ( FIVEDIT.INI ), y si no existe lo crea

Ejemplo: Fivedit
Ejemplo: Fivedit myprog.prg
Ejemplo: Fivedit User1.ini
Ejemplo: Fivedit myprog.prg User1.ini

Recomiendo crear un acceso directo con el nombre del usuario que va a utilizar el Editor, si no la utilización de la aplicación
tomará el fichero de configuración genérico, que ha podido ser modificado por otro usuario.

Así:
Ejemplo: Fivedit User1.ini

Usted puede copiar el fichero genérico al nombre que usted decida para poder utilizar la configuración existente

[ ENGLISH ]

You can create a shortcut to the .EXE file (Fivedit.EXE) as it normally does, but the program accepts two parameters:

1.- Source program (.PRG) to open directly
2.- Configuration file (.INI), if none is indicated, the configuration file has the name of the .EXE,
     In this case (FIVEDIT.INI), and if it does not exist create it

     Example: Fivedit
     Example: Fivedit myprog.prg
     Example: Fivedit User1.ini
     Example: Fivedit myprog.prg User1.ini

     I recommend creating a shortcut with the name of the user who will be using the Editor, if not using the application
     Will take the generic configuration file, which has been modified by another user.

This:
     Example: Fivedit User1.ini

You can copy the generic file to the name that you decide to be able to use the existing configuration

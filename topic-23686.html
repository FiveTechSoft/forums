Yo trabajo desde hace mas de un año con ado y funciona de maravilla, he hecho aplicaciones con mysql, sql server (2005 y 2008) y tablas de access .

es algo completamente distinto a trabajar con dbf, ya que debes conocer sentencias sql (que son poderosisimas), te recomiendo que sigas esta opcion ya que no tienes limites puedes hacer clase de scripts, en mi experiencia trabajo mucho con procedimientos almacenados en el servidor.

yo me adapte super rapido porque tenia cierta experiencia con instrucciones sql y tenia instalado el microsoft sql management console, que te permite probar todos los scripts y practicar programacion en sql, en internet hay millones de ejemplos, guias, documentacion, etc.

por ejemplo para ver una tabla con ado, debes enviar un script al servidor como por ejemplo "select * from clientes" y te devuelve un recordset con todos los campos y todos los clientes.

cMa_Func:= tOleAuto():New("ADODB.Recordset")
cMa_Func:CursorLocation(3)
cMa_Func:Open("SELECT * FROM ma_func WHERE ma_clave='"+ AllTrim(cCodVend) +"' ORDER BY ma_clave", oConexion, 1, 3) //el 2 := cursor dinámico
If (cMa_Func:RecordCount) == 0
cNomVend:="No Existe Funcionario"

Else
cNomVend:=ADOField(cMa_Func,"ma_nomf")
Endif
cMa_Func:Close()

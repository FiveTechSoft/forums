Estimado Joao,



No, la lógica no es correcta. Hay varios problemas:



  • 1.

    oPrn:hDC no es un código de estado de impresora



    hDC es el Device Context handle de Windows. Después de PRINT oPrn, su valor será:


  • -

    0 → falló la creación (impresora no disponible)


  • -

    Un número grande (ej. 0xA10045B2) → handle válido, la impresora está lista



    Nunca será 1, 2, 3 o 4 como códigos de estado. Esas comparaciones nunca se cumplirán.



  • 2.

    La estructura PRINT...ENDPRINT está rota



    PAGE/ENDPAGE queda fuera del flujo cuando haces RETURN NIL dentro del bloque PRINT...ENDPRINT. Si hDC != 0, el código

    nunca llega a PAGE porque siempre entra en algún ELSEIF.



    Versión corregida (simple):



    • Code: Select all Collapse
    </s>  FUNCTION IMPR_ATIVA()

<i>    </i> PRINT oPrn NAME "Teste"

<i>    </i>    IF oPrn:hDC == 0
<i>    </i>       MsgStop( "Impressora desligada ou não encontrada!" )
<i>    </i>       RETURN NIL
<i>    </i>    ENDIF

<i>    </i>    PAGE
<i>    </i>       oWnd:Print( oPrn, 1, 1, 2 )
<i>    </i>    ENDPAGE

<i>    </i> ENDPRINT

  RETURN NIL<e>


    Si necesitas el estado real de la impresora, usa la API de Windows GetPrinter() o en FiveWin:



    Code: Select all Collapse
    </s>  FUNCTION IMPR_ATIVA()
     LOCAL cPrinter := GetDefaultPrinter()
     LOCAL nStatus  := GetPrinterStatus( cPrinter )  // necesitas implementar via API

<i>    </i> // Los códigos reales de Windows (PRINTER_STATUS_*) son:
<i>    </i> // PRINTER_STATUS_PAUSED       := 0x00000001
<i>    </i> // PRINTER_STATUS_ERROR        := 0x00000002
<i>    </i> // PRINTER_STATUS_PAPER_OUT    := 0x00000010
<i>    </i> // PRINTER_STATUS_OFFLINE      := 0x00000080
<i>    </i> // PRINTER_STATUS_PRINTING     := 0x00000400

<i>    </i> IF nStatus == 0
<i>    </i>    // Status OK, imprimir
<i>    </i>    PRINT oPrn NAME "Teste"
<i>    </i>       PAGE
<i>    </i>          oWnd:Print( oPrn, 1, 1, 2 )
<i>    </i>       ENDPAGE
<i>    </i>    ENDPRINT
<i>    </i> ELSE
<i>    </i>    MsgStop( "Impressora com problema! Status: " + Str(nStatus) )
<i>    </i> ENDIF

  RETURN NIL<e>


    Resumen: Para una verificación básica, basta comprobar oPrn:hDC == 0 (fallo) vs. distinto de cero (OK). Los valores

    1-4 como estado de impresora no tienen relación con hDC.

    Estimado Joao,



    Revisado con Claude code:



    1. IF invertido → ahora verifica nStatus != 0 para detectar problemas, y solo imprime si todo está OK


  • 2. == reemplazado por lAnd() → los status son bitmask, se deben comprobar con AND de bits

  • 3. PrnStatus() llamada una sola vez → el resultado se guarda en nStatus

  • 4. isprint() eliminada → no es función estándar de FiveWin

  • 5. Más estados cubiertos → paper jam, paused, no toner, toner low, door open, user intervention

  • 6. Flujo limpio → primero valida, si hay problema sale con RETURN NIL, si no continúa a imprimir


    • prnativa.prg

Excelente Roberto. Por favor mantenme al tanto de tus pruebas.

Por mi parte te informo que hice un pequeño codigo en Python para probar el lector de codigo de barra y este funciona perfecto y sin ningun problema de lentitud.

Antonio,
Parece que lo que hace mas lento a la Raspberry Pi es el uso de la SD Card como Hard Drive. Debieramos hacer algunas pruebas usando como almacenamiento de datos las "nubes" (cloud).
Ademas es logico suponer, dado el apoyo que tiene, que por ahi vendra un Raspberry Pi mas rapido y con mas RAM memory.

Por otro lado Harbour + FiveWin nos permite usar codigo C lo que nos pemitiria escribir algunos modulos que funcionen mas rapido.

Otra opcion, para hacer que nuestros programas corran mas rapido en la Raspberry Pi, es usar ' Assembly Language' (solo para algunos modulos que requieran de una mayor velocidad) a traves de BBC BASIC.
Esto no parece muy dificil ya que el BBC BASIC, a traves del cual se llaman las instrucciones assembler, se asemeja al lenguaje BASIC que aprendimos decadas atras.
Ya hay un libro sobre este tema denominado "Raspberry Pi Assembly Language" que lo podemos adquirir en amazon.com

Asi que no nos desanimemos y sigamos haciendo testing.

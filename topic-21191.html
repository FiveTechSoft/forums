New FTDN March/Marzo 2011 (FWH 11.03)

Posted: Mon Mar 28, 2011 10:23 AM

March 2011

  • Enhancement: Class TXBrowse (f) In fast edit mode pressing '.' also triggers edit of numeric data.

  • Fix: xbrowse.prg (c) SortArrayData: After sorting an array, the selected row is positioned on the same
    row before sort

  • Enhancement: Class TSay: unlimited text length in REDEFINE SAY

  • Enhancement: Class TRBTN now we can use codeblock to define lines color ( in line / out line ),
    codeblock argument is lMover, mouse over status

  • Fix: Class RBTN, now the back color is correctly painted, reported by mgsoft:

viewtopic.php?p=112249#p112249

  • Fix: Memory Leak, some GDI object are not released correctly in dialog, now is ok
    reported by IBTC: viewtopic.php?p=111054#p111054

  • Fix: Class Tbtnbmp.prg fixed suggested by Enrico

viewtopic.php?p=110529#p110529

  • Enhancement: Class TXBrowse.prg: method Resize() now ends with Refresh()

  • New: function fw_decode() in valblank.prg: as replacement to buggy hb_decode() function in (x)Harbour

  • Enhancement: samples/wordtable.prg: revised to work with Harbour also

  • Enhancement: Class TImage Method LoadFromMemory( cBuffer, nWidth, nHeight ) now it supports two optional
    parameters to resize the image to a desired width and height. Many thanks to Ruediger!

  • New: Method RButtonDown( nRow, nCol, nFlags ) CLASS TMsgBar and Method METHOD RClick( nRow, nCol ) in
    Class TMsgItem to support right clicks on msgbar items

  • Fix: Some memory leaks in Class TGraph. Thanks to Ruediger!

  • Enhancement: Class TRibbonBar double click and right click at the quickbutton support. Thanks to Ruediger!

  • Fix: bug reported in viewtopic.php?f=3&t=20807&p=111080
    New function FUNCTION RefreshRibbonBarBtn( oRb ) to be called everytime when bWhen is changed.
    Thanks to Ruediger!

  • Fix: Memory leak in Class TDialog

  • Fix: Clause 2007 was releasing a bitmap but not removing it from the FWH list. Now it is ok.
    (still a pending one there).

regards, saludos

Re: New FTDN March/Marzo 2011 (FWH 11.03)

Marzo 2011

  • Mejora: En la clase TXBrowse (f). En el modo de edición rápida, pulsando '.' también se dispara la
    edición de los datos numéricos.

  • Corrección: En xbrowse.prg (c). SortArrayData: Después de la ordenación de una matriz, la fila seleccionada
    se posiciona en la misma fila antes de la ordenación.

  • Mejora: En la clase TSay: Se puede utilizar textos de longitud ilimitada en REDEFINE SAY.

  • Mejora: En la clase TRBTN, ahora podemos usar un bloque de código para definir las líneas de color
    ( in line / out line ), el argumento del bloque de código es lMover.

  • Corrección: En la clase RBTN, ahora el color de fondo se pinta correctamente, reportado por mgsoft:
    viewtopic.php?p=112249#p112249

  • Corrección: Pérdida de memoria, algunos objetos GDI no se liberan correctamente en los diálogos, ahora sí,
    reportado por IBTC: viewtopic.php?p=111054#p111054

  • Corrección: En el fichero Tbtnbmp.prg, las correcciónes sugeridas por Enrico viewtopic.php?p=110529#p110529.

  • Mejora: En el fichero TXBrowse.prg, el método Resize() ahora finaliza con Refresh().

  • Nuevo: Función fw_decode() en valblank.prg: como reemplazo a la función errónea hb_decode() en (x)Harbour.

  • Mejora: El ejemplo samples/wordtable.prg: revisado para trabajar con Harbour también.

  • Mejora: En la clase TImage, el método LoadFromMemory( cBuffer, nWidth, nHeight ) ahora soporta dos
    parámetros para redimensionar la imagen a un ancho y altura deseadas. Muchas gracias a Ruediger!

  • Nuevo: Los métodos RButtonDown( nRow, nCol, nFlags ) de la clase TMsgBar y el método RClick( nRow, nCol )
    en la clase TMsgItem para soportar el click en el botón derecho del ratón sobre los items de la barra de mensajes.

  • Corrección: Alguna pérdida de memoria en la clase TGraph. Gracias a Ruediger!

  • Mejora: En la clase TRibbonBar, doble click y click en el botón derecho del ratón en el soporte de botón rápido.
    Gracias a Ruediger!

  • Corrección: Error reportado en viewtopic.php?f=3&t=20807&p=111080
    Nueva función RefreshRibbonBarBtn( oRb ) que se llamará cada vez que se cambie bWhen. Gracias a Ruediger!

  • Corrección: Pérdida de memoria en la clase TDialog.

  • Corrección: La claúsula 2007 liberaba un bitmap pero no lo quitaba de la lista de recursos en uso que mantiene FWH. Ahora está correcto.
    (aúm queda uno pendiente de liberar).

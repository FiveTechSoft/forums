En Clipper DOS es posible, nosotros desarrollamos una tecnologia propietaria para accesar a bases de datos de MySQL via Clipper 5.3, puro y duro, y claro, un poco de ayuda del interprete de MySQL, al final de cuentas funciono muy bien, mejor de lo que esperabamos pero si, es un poco lento.

Con FiveWin 16 bits, tambien desarrollamos varias aplicaciones, principalmente con las clases TODBC y TDBODBC, principalmente para Oracle y para Informix, adicionalmente FW trae el RDDODBC, que si bien no es la forma mas optima de acceder a datos SQL funciona bien.

En el mundo de los 32 bits hay muchas mas herramientas, nosotros preferimos ADO, mas que nada por la facilidad de moverte entre distintos motores de base de datos con el mismo codigo fuente y cambiando solo una linea de codigo.

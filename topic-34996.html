If you want to use ADO, then FW_Ado... functions provide a good support. The FWH's ADO library helps you to use ADO with Access, MSSQL, MySql, Oracle, etc. without much change in the code.

If you use ADO with MySql or Oracle, etc you need to ensure installation of ODBC or ADO provider on each PC running your program.

If you propose to migrate to MySql/MariaDB then FWH natively provides the required support. You do not require any other libraries, though 3rd party alternatives are available. FWH MariaDB/MySql libraries are meant for novices as much as advanced users and designed to make migration easier.

In this case, you need to distribute one DLL along with your EXE and requires no other installation. This is the easiest option.

Our advice is to consider MariaDB

You require "libmysqld.dll" only if you want to use "embedded" MySql server. For connecting to normal MySql/MariaDB server on local PC or a network server, you have all the files available in fwh distribution.

If you want to try Embedded Server, please download sample from http://anserkk.com/gnraomysql/view.php?id=15. The downloaded zip file contains all files required including libmysqld.dll.

Frances

I was curious if SqlOleDb was still supported with Sql Server 2012 so I went out to downloaded the Sql Express version and loaded it on my laptop.

Much to my surprise, I was pleased to find that Sql Server 2012 still supports the legacy ADO provider. I will say that Microsoft is pushing their own native Ole client SQLNCLI11, Dot net and ODBC .. see the supported providers below:

http://connectionstrings.com/sql-server-2012

As you can see .. there is no mention of SqlOleDb, however it still connects and works quite well from Windows 7. I am sure it is just a matter of time until Microsoft completely diminishes SqlOleDB in favor of Dot Net and ODBC .. but with the continued support of Sql Server 2012 .. legacy 32 bit support ( native to all windows ) will be around for a few more years

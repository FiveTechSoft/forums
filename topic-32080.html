Ruben,

Con FiveWeb hay procesos que se ejecutan en el servidor y procesos que se ejecutan en el navegador.

Cuando tu intentas ejecutar esa función, la aplicación ya esta funcionando en el navegador y ahi no
existe esa función salvo que llames nuevamente al CGI de FiveWeb.

Entender esa diferencia es la clave de entender FiveWeb

Hola :

Mira este ejemplo, se va autollamando para ejecutar funciones, en algún punto tambien llama a otro exe.

https://dl.dropboxusercontent.com/u/496 ... sional.prg

Cuando se autollama se pierde todo ( Eje. variables static , tablas abiertas .. ) por eso hay que hacer nuevamente la conexión al diccionario o las aperturas de tablas.. etc ... es como que estaría llamando a otro exe .

Solo se pasa parametros de un EXE o otro..

Si trabajas con el chrome tenes la posibilidad de habilitar para ver como se va ejecutando tu soft, en configuración del chrome -> mas herramientas -> herramientas para desarrolladores..
También si te paras sobre tu soft en ejecución y presionas el botón derechos puedes analizar Ver código fuente de pagina..

Jose:



jose_murugosa wrote:
1) Si deseo utilizar ADS REMOTE SERVER 8.0, tendré que instalar un software como Windows 2003 server o un server de linux, o es posible continuar con el grupo de trabajo que ya usan?


Aunque el uso de ADS te sugiere que tengas un servidor dedicado, nosotros hemos hecho instalaciones de ADS sobre Windows XP Professional (muy importante que sea PROFESSIONAL), donde el servidor de datos a su vez es estación de trabajo, y en redes pequeñas hasta para 5 usuarios funciona estupendamente bien, para redes con mas de 5 usuarios, si es mas recomendable un equipo con hardware y software de servidor.


jose_murugosa wrote:
2) He visto que ADS es fabuloso, pero algo caro, por lo que me interesa saber... cuando se habla de 5 licencias que significa exactamente..
Esta empresa tiene 10 equipos, todos usan el mismo sistema (el mismo exe) pero difícilmente mas de 3 equipos ocurren a las mismas bases de datos simultáneamente (jamás más de 5).
Puedo usar ADS con 5 licencias y no habrá problema o debo comprar 10 licencias por usar el mismo software en los 10 equipos?


5 licencias significan USUARIOS CONCURRENTES es decir, estaciones de trabajo, que, sin importar el número de aplicaciones que hagan uso de las tablas en el servidor, están accediendo a los datos en ese momento.

Por ejemplo, mencionas que te tienes 10 puestos pero que aunque los 10 usan tus programas NO LO HACEN AL MISMO TIEMPO, si este es el caso, una licencia de 5 usuarios te funciona perfectamente, la limitante es que un 6to usuario tendra que esperar a que se "libere" una licencia para poder utilizar el programa, de hecho tengo un caso precisamente en este momento de un cliente que tiene 5 licencias y 15 usuarios y todos usan el programa, los empleados tienen que "esperar turno" para poder usar el programa o tienen que pedirse "prestados" los usuarios es decir, esperar a que se desconente uno para que otro pueda acceder al servidor.

jose_murugosa wrote:
3) El "Local Server de ADS", según entiendo NO ES CLIENTE/SERVIDOR , pero permite desarrollar por lo que he leído la aplicación y dejarla lista para colgarla en un server ADS REMOTO.
¿Hay otras ventajas en usar este "Local Server"?, según entiendo con él podemos tener hasta 5 usuarios ocurriendo a la misma base de datos, pero... no estaremos a salvo de corrupciones, verdad?


Así es, con hasta 5 usuarios, el servidor local se comporta muy bien, pero careces de 4 cosas importantes contra el servidor remoto:

1) No es cliente servidor, lo que quiere decir que continuas sujeto a los errores típicos de las bases de datos DBFCDX
2) No tiene soporte a transacciones
3) No tienes protección por ocultamiento de las carpetas donde están los datos (aunque si tienes encriptación)
4) No tienes acceso via internet.

El ADS Local Server no está pensado como un producto para aplicaciones comerciales para redes, aunque muchas empresas lo usan así, por eso es de distribución gratuita, el ADS Local Server tendría que considerarse mas bien como una herramienta para desarrollo y pruebas de software, que le permite al programador usar todas las características de ADS y probar su software en redes pequeñas o en monousuario, evitando con ello tener que estar "pegado" a un servidor que usualmente está metido en una oficina, digamos que el servidor local es un ADS "para llevar" en el portatil y poder programa en casa y probar todas las características del programa mismas que serán utilizadas en un servidor remoto el lunes siguiente.

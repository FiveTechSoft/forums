Jorge,

Hasta donde yo sé, la única forma de abrir tablas con ads remoto es usando sus librerías.
Ya que ads toma control sobre las tablas.

Si no tienes ninguna tabla abierta con las librerías ADS debería funcionar, pero en el momento que abras alguna dejará de funcionar.

ADS server es un sistema cliente-servidor con las muchisimas garantias que esto ofrece, por lo tendrá que bloquedar el acceso externo a las tablas para evitar errores.

El problema es que hasta la fecha no existe librerias ADS para pockect. Ofrecen librerías para web con lo que ya se puede acceder desde cualquier sitio.

Si quieres seguir trabajando contra las tablas ADS remoto directamente sin tablas intermedias, mi consejo es que pruebes Terminal Server.

Jorge,

ADS ofrece liberías para programas web.

Mi consejo es que compruebes si puedes usar Microsoft Terminal Server. (Necesitas un servidor con licencias terminal server)
De esta forma, los dispositivos pocket, no tendrían una aplicación propia instalada en ellos. Lo que tendrían sería un acceso via terminal (RDP, al servidor donde, al conectarse, se ejecutaría la aplicación.

Esta aplicación, al estar instalada en el servidor, ya no sería necesario que fuera para pocket, sería una aplicación para windows normal con su ADS, Osea en lugar de usar FWPPC usar FWH (Limitado, claro esta, al tamaño máximo que pueda soportar el dispositivo movil.

Sobre gustos no hay nada escrito y puede que esta solución guste mas o menos, pero funciona.

Así tengo funcionando varias maquinas y hasta la fecha no he tenido ningún problema.

    ELSE

    ENDIF

You need a mobile phone for this to work!





### 📱 **Do you need a mobile phone for this to work?**

**Yes, you do.** The FiveWin program you've shown works by launching WhatsApp either through the **WhatsApp Desktop App** or the **WhatsApp Web API**, and both require a **linked mobile phone with WhatsApp installed and active**.

---

### 🔗 How the system works

The code uses one of two methods to send a message:

#### 1. `https://api.whatsapp.com/send?phone=...&text=...`

* Opens **WhatsApp Web** in your browser.
* Requires the user to have previously scanned a QR code using their phone to link it with the browser session.
* A real phone with WhatsApp is needed for this to work.

#### 2. `whatsapp://send?phone=...&text=...`

* Opens the **WhatsApp Desktop App** (if installed).
* Again, this desktop app must be linked to a **real phone via QR code pairing**.

---

### ✅ What does WhatsApp provide?

* The “clearing” or delivery mechanism is **WhatsApp itself**, via the URL scheme or API entry point.
* There is **no external clearing server** involved.
* WhatsApp ensures that messages are sent from a real phone account – it **does not allow sending messages without a linked mobile device**.

---

### 🔄 Can it be automated?

**No, not fully**, unless you use the **WhatsApp Business API**. Here's a comparison:

| Method | Requires Phone | Fully Automated | Easy Setup | For Business Use |
| ------------------------------- | ----------------------------------- | --------------- | ---------- | --------------------------- |
| `https://api.whatsapp.com/send` | ✅ Yes | ❌ No | ✅ Yes | ✅ Personal / Small Business |
| `whatsapp://send` | ✅ Yes | ❌ No | ✅ Yes | ✅ Personal / Small Business |
| WhatsApp Business API | ❌ No (but requires official number) | ✅ Yes | ❌ Complex | ✅ Medium / Large Business |

---

### ⚠️ Important Notes

* The number must be in **international format** (e.g. `5492324...` for Argentina).
* The phone must be **online** and have WhatsApp active.
* Users will usually need to **click “Send”** manually after the message is opened, unless advanced automation tools are used (which are often against WhatsApp's terms).

---

### 💡 Summary

You **cannot send WhatsApp messages without a real phone** — whether using the browser, the desktop app, or even via automation. WhatsApp's infrastructure requires all messages to go through a **verified phone account** unless you use the official Business API with approval and a provider like Twilio or 360dialog.

Would you like a version that cycles through a list and opens WhatsApp for each contact with a delay and user guidance? I can create that as well.


Great question — and yes, **some degree of automation is possible**, *but only partially and with important limitations*.

---

### ✅ **What you *can* automate using WhatsApp Desktop + Timer**

You can:

* Automatically **launch the WhatsApp Desktop app** using `ShellExecute`.
* Automatically **open a chat with a specific phone number** using `whatsapp://send?phone=...&text=...`.
* Use a **timer or delay** (`Sleep()`) between launches.
* Potentially **simulate user input** (like keyboard events) via:

* Windows API (e.g., `keybd_event`, `SendInput`),
* AutoHotKey scripts,
* `INPUT` struct via low-level Windows automation.

---

### ⚠️ **What you *cannot* reliably automate**

WhatsApp Desktop is:

* **Not designed to be automated** (there is no official API for the desktop app).
* Subject to **UI changes** (button positions, window titles, etc.).
* Dependent on **human interaction** — WhatsApp intentionally makes it difficult to *fully* automate sending messages to prevent spam and abuse.

Also:

* You **cannot detect if the message was actually sent**.
* If the chat is already open, `whatsapp://send?phone=...` won't pre-fill the message reliably.
* **Simulating the Enter key to send** is fragile (it may send the wrong message or fail if focus is lost).

---

### 🧪 ✅ Feasible hybrid solution for semi-automation

You can build a **loop with a timer** like this:

```xbase
STATIC FUNCTION LoopSendWhatsApp( aData, oDlg )
LOCAL i, cTexto, lWhatsapp_escr := .t.

FOR i := 1 TO LEN(aData)
cTexto := "Hello *" + aData[2] + "* from *" + aData[3] + "*!"
cTexto := STRTRAN(cTexto, " ", "%20")
cTexto := STRTRAN(cTexto, CHR(13), "%0D%0A")

ShellExecute( oDlg:hwnd, "open", ;
"whatsapp://send?phone=" + ALLTRIM(aData[1]) + "&text=" + cTexto )

Sleep(3000) // wait before next
NEXT
```

Then combine it with user instructions like:

> *"Click 'Send' manually for each message. The next message will open in 3 seconds."*

---

### 🛠 Advanced Option: AutoHotKey + WhatsApp Desktop

With [AutoHotKey](https://www.autohotkey.com/) you can:

* Wait for the WhatsApp window.
* Send keys (`Send Hello!{Enter}`).
* Close the window, go to the next.

But this is brittle and **not officially supported**.

---

### ✅ Summary

| Approach | Fully Automated? | Stable? | WhatsApp-compliant? | Notes |
| ------------------------------------- | ---------------- | --------- | ------------------- | ------------------------- |
| WhatsApp Web / Desktop + Timer | ❌ Partial | ✅ Yes | ✅ Yes | Requires user interaction |
| Simulated keyboard/mouse (AutoHotKey) | ✅ Mostly | ❌ Fragile | ❌ Risky | Can break on updates |
| WhatsApp Business API | ✅ Yes | ✅ Yes | ✅ Official | Paid, complex setup |

FUNC SendToWhatsApp( cPhone, cMsg, aAttach )
LOCAL oShell, aFiles := {}, aOthers  := {}, lRet
 If (lRet := !Empty( cPhone ))
  If !Empty( aAttach )
   If( !HB_ISARRAY( aAttach ), aAttach := { aAttach }, )
   AEval( aAttach, {|cFile| If( HB_ISSTRING( cFile ) .AND. File( cFile ), AAdd( aFiles, cFile), AAdd( aOthers, cFile ) ) } )
  EndIf
  cMsg  := StrTran( cMsg, CRLF, "%0D%0A" )
  oShell := CreateObject( "WScript.Shell" )
  ShellExecute( 0, "Open", "whatsapp://send?phone="+cPhone+"&text="+cMsg)
  SysWait( 0.2 )
  If !Empty( aFiles )
   If FW_1CopyToClipBoard( aFiles, 15 )
    SysWait( 2 )
    oShell:SendKeys( "^v" )
    SysWait( 1 )
   EndIf
  EndIf
  AEval( aOthers, <|u|
                      If FW_1CopyToClipBoard( u, 2 ) .OR. FW_CopyToClipBoard( u )
                       SysWait( 1 )
                       oShell:SendKeys( "^v" )
                       SysWait( 1 )
                      EndIf
                      RETURN NIL
                  > )
   oShell:SendKeys("~")
 EndIf
RETURN lRet

function FW_1CopyToClipBoard( uValue, nFormat )

   local lCopied  := .f.
   local aBmp

   if HB_ISPOINTER( uValue ) // assumes GDI+ Image object
      GDIPLUSImageToClipBoard( uValue, GetDesktopWindow() )
      return .t.
   endif

   if uValue != nil

      if nFormat == 15
         if !HB_ISARRAY( uValue ); uValue := { uValue }; endif
         _fwhtmp_ := uValue
         uValue   := {}
         AEval( _fwhtmp_, { |c| If( File( c ), AAdd( uValue, TrueName( c ) ), nil ) } )
         _fwhtmp_ := nil
         if Empty( uValue )
            return .f.
         endif
      endif

      if Empty( nFormat ) .and. HB_ISARRAY( uValue ) .and. ;
         AScan( uValue, { |u| !( ValType( u ) == "C" .and. File( u ) ) } ) == 0
            AEval( uValue, { |c,i| uValue[ i ] := TrueName( c ) } )
            nFormat  := 15
      endif

      if nFormat == 2 .and. HB_ISSTRING( uValue )
         aBmp  := FW_ReadImage( nil, uValue, , .f. )
         if !Empty( aBmp[ 1 ] )
            uValue := aBmp[ 1 ]
         else
            return .f.
         endif
      endif

      if Empty( nFormat )
         if HB_ISSTRING( uValue )
            if IsBinaryData( uValue )
               if Left( MemoryBufferType( uValue ), 4 ) == "IMG."
                  aBmp := FW_ReadImage( nil, uValue, , .f. )
                  if !Empty( aBmp[ 1 ] )
                     uValue   := aBmp[ 1 ]
                     nFormat  := 2
                  endif
               endif
               if Empty( nFormat )
                  return .f.
               endif
            elseif Left( uValue, 4 ) == "http"
               aBmp := FW_ReadImage( nil, uValue, , .f. )
               if !Empty( aBmp[ 1 ] )
                  uValue   := aBmp[ 1 ]
                  nFormat  := 2
               else
                  nFormat  := 1
               endif
            else
               nFormat  := 1
            endif
         elseif HB_ISNUMERIC( uValue )
            if ISHBITMAP( uValue )
               nFormat := 2
            elseif ISENHMETA( uValue )
               nFormat  := 14
            endif
         endif
      endif

      if Empty( nFormat )
         if ValType( uValue ) $ "AH"
            uValue   := FW_ValToExp( uValue )
         elseif ValType( uValue ) $ "DT"
            uValue   := CharRem( "'", FW_DateToSQL( uValue ) )
         else
            uValue   := cValToChar( uValue )
         endif
         nFormat  := 1
      endif

      if nFormat != nil
         if nFormat == 2
            if HB_ISNUMERIC( uValue ) .and. ISHBITMAP( uValue )
               GDIPLUSHBITMAPTOCLIPBOARD( uValue, GetDeskTopWindow() )
               lCopied  := .t.
            endif
         elseif OpenClipBoard( GetDeskTopWindow() )
            EmptyClipboard()
            if nFormat == 1 .and. IsUtf8( uValue )
               uValue    := utf8toutf16( uValue )
               if !( Right( uValue, 2 ) == Chr( 0 ) + Chr( 0 ) )
                  uValue    += ( Chr( 0 ) + Chr( 0 ) )
               endif
               lCopied  := SetClipboardData( 13, uValue )
            elseif nFormat == 15
               lCopied  := SetClipboardData( 15, uValue )
            else
               lCopied  := SetClipboardData( nFormat, uValue )
            endif
            CloseClipboard()
         endif
      endif
   endif

   if aBmp != nil
      PalBmpFree( aBmp )
   endif

return lCopied

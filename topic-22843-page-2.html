yo tengo como 2 aos de usar el resedit el cual tiene version de 32 y 64 bits, sin problemas, es el mejor cambio que pude haber echo, la verdad no echo de menos el resource workshop.

aca lo importante es la configuracion de las rutas donde estan los ficheros cabezeras del sdk del compilador c.

lo que sucede, es que al intentar pasar nuestro archivo rc o res directamente desde ws a resedit, va a dar algunas fallas, ya que no encuentrara algunas definiciones de constantes del api de controles o cajas de dialogos. esto por que el ws no incluye de manera automatica los ficheros cabezeras estandar del sdk de windows, ya que el ws reconoce la mayoria de las constantes, aun cuando no le indiquemos estos archivos cabezeras.
por ejemplo incluir el

include <windows.h>

include <commctrl.h>

include <richedit.h>

etc.

en la configuracion del resedit podemos ver cuales son los que el por defecto incluye cuando se crea uno nuevo, para que nosotos incluyamos estos mismos de forma manual en los nuestros antes que el resedit intente abrirlos.

tiene sus failliats como por ejemplo que los combobox no tiene el tab stop definido por defecto, hay que cambiarlo a true en las propiedades del control.

Bueno, comentarles que trabajando con marcelo
se ha realizado lo siguiente.-

1.-instalación de virtualbox. ( virtualbox.org )
2.-instalación de ubuntu en virtualbox (se uso una conexión de tipo "red puente" para que la pc virtual tome ip del mismo router, no haciendo NAT desde la pc anfitrión), quedando.
a.-pc física( windows 7 ) con ip 192.168.2.101
b.-pc virtual (ubuntu) con ip 192.168.2.103
(ambas responden ok a un ping entre ellas.)
3.-instalación de subversion, ncurses5, gpm, etc. para poder construir el compilador harbour.
4.-se bajo los fuentes de harbour svn en una carpeta propia. ( /home/user/harbour_svn )
5.-creación de carpeta para cuando se construyan los binarios de harbour, y creando la variable HB_INSTALL_PREFIX=/home/user/harbour_bin.
6.-construcción de harbour. ( /home/user/harbour_svn/harbour/make install )
7.-verificación del compilado harbour, construyendo la super aplicación "hello word!" usando hbmk2 ( /home/user/harbour_bin/harbour/bin/hbmk2 /home/user/harbour_svn/harbour/test/helloword.prg )
8.-se bajo el zip que contiene los fuentes de fiveweb. ( se descomprimió una carpeta llamada /home/user/fiveweb_src )
9.-se modifico las rutas del makefile de fiveweb (/home/user/fiveweb_src/makefile ) para poder recompilarlo, apuntando a los binarios de harbour. (/home/user/harbour_bin)
10.-se construyo la librería libfiveweb.a usando make en la carpeta raiz de fiveweb_src
11.-para comodidad se movió la lib recién creada a /home/user/harbour_bin/lib
12.-se ha modificado los path de build.sh en /home/user/fiveweb/samples para que apunten los binarios de harbour, asi como los includes de los binarios.
13.-se ha creado tutor01 usando build.sh exitosamente.
14.-se ha instalado apache en ubuntu
15.-se ha validado que en el navegador de ubuntu usando "localhost" veamos el servidor apache.
16-se ha validado en el navegador de windows 7 usando "192.168.2.103" que veamos el servidor apache de ubuntu.

hasta ahí vamos, y todo nos ha salido ok.

falta.

1.-instalar un mysql y php. (no se si sera necesario para las pruebas.)
2.-configurar apache para permitir cgi-bin
3.-poner el ejecutable tutor01 en la carpeta de los cgi-bin
4.-probarlos en los navegadores.

ya aca es cuando, para no reinventar la rueda, preguntamos si alguien tiene un poco de experiencia en esto ultimo.

esta es la parte que mas odio y amo de harbour.
que para hacer "ciertas" cosas tengas que aprender un montón de cosas. ( c, make, linux, terminal, redes, ssh, web, etc., por mencionar algunas cosas )

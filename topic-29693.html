Harbour is about data processing and in this list there is surely a
huge amount of knowledge about accounting, manufacturing, CRM e so on.

We all use Harbour language but we use different data structures that
very likely contain more or less the same data: customers, orders,
invoices, transactions, bill of materials and so on. The problem is
that the field's name is different, table's name is different and
probably they have a meaning in our local language but that are
meaningless in other languages.

Let's imagine for a moment that we'll work together to create a common
data dictionary where we define tables, fields, relations and create
few command line tools that can create those tables in dbf and sql and
that can upload/convert data from our proprietary data structure.

Let's imagine also to collaborate to UI translation. I need Italian
but I also need English and German . We could create a central repo
for bit files and exchange them.

Designing a multi language/multi currency data dictionary, means also
exchange knowledge about local requirements and tax regulations. For
example Europeans could contribute info for VAT or INTRASTAT while
others may contribute info in other areas.

Let's imagine to "sell/buy" entire modules or even entire
applications. For example I can buy an accounting application that is
ready for Brazilian market since I can easily integrate it with my CRM
.

I understand that it is a long term goal, but I think this will give
value to Harbour's ecosystem. And even if we will not achieve concrete
and usable results, the info exchanged would not be lost.

What do you think?

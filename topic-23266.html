Hace tiempo migré mis aplicaciones a MySql. Me incliné por Ado ya que me parecía que de esta manera iba a poder aprovechar más la potencia del motor. Para comenzar leí el blog de Armando Estrada Bucio ( http://www.sqlcmd.blogspot.com ), allí pude encontrar explicaciones muy claras y con ejemplos sencillos muy fáciles de entender ( Gracias Armando ), después googlendo encontré infinidad de sitios sobre el tema. Como dice Daniel, el problema no es aprender Sql ( que es muy sencillo ) sino tener otra mirada sobre la manera de trabajar con BBDD. La posibilidad de programar el motor, el roolback, olvidarnos de los bloqueos a registros, incorruptibilidad de los índices, los triggers y las store procedure y un montón de ventajas más dan una potencia y seguridad incomparables. Tengo una aplicación de liquidación de sueldos trabajando con 50 usuarios concurrentes y prácticamente no necesita mantenimiento. Creo que la elección de base de datos planas .dbf ya no es válida salvo para pequeñas aplicaciones que requieran poca exigencia.

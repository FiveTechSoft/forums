Best Practices for modern Web/DBF integrations: Cloudflared, Mollie, MessageBird?
Posted: Sat Mar 07, 2026 08:53 AM

Hello friends,



In our team, we are currently modernizing our Harbour/DBF backend to better integrate with the modern web (Omnichannel messaging & seamless payments) while strictly keeping our DBFs and business logic safe and on-premise (GDPR compliance).



To avoid reinventing the wheel and writing dozens of custom API integrations, we are thinking about adopting a "standardized stack" of European API providers. The goal is to build our Harbour code against these APIs once, and ideally share our experiences (and code snippets) with the community.



This is the architecture we are currently planning to implement:



  • 1.

    Security & Webhooks -> Cloudflared (Argo Tunnel) Instead of opening local ports on our routers to receive Webhooks from external APIs, our local Harbour server connects outbound to a Cloudflare Tunnel. This means our local DBFs are completely shielded from the public internet, but we can securely receive JSON Webhooks (e.g., from payment providers).



  • 2.

    Payment Gateway -> Mollie Instead of touching credit card data (and dealing with PCI-DSS compliance), we want to use the Mollie API. We create a payment link in Harbour -> send it to the guest -> Mollie handles the rest (Credit Card, Apple Pay, SEPA) -> Mollie sends a secure Webhook back through Cloudflared to our server, and Harbour updates the BELEGUNG.DBF to "PAID".



  • 3.

    Communication (WhatsApp/SMS) -> MessageBird We are looking to use MessageBird (a European alternative to Twilio) to send automated pre-arrival WhatsApp messages or SMS directly from our Harbour logic. Incoming replies from guests trigger a Webhook that Harbour saves directly into the guest's HISTORY memo field.



  • 4.

    Transactional E-Mails -> Brevo (formerly Sendinblue) Sending booking confirmations via REST API instead of dealing with complex SMTP setups in Harbour.



    • My questions to the community:



    Is anyone here already using Cloudflared tunnels to easily receive incoming Webhook-JSONs straight into a Harbour-compiled microservice?

    Has anyone already written integration classes for Mollie or MessageBird/Brevo in Harbour that they could share, or would you be interested in joining forces to build a stable class for these specific providers?

    I believe that if some of us agree on a shared stack of standard providers like these, we can save a tremendous amount of time maintaining different API endpoints.



    Looking forward to your thoughts and experiences!



    Best regards, Otto

FUNCTION handle_inbound_msg( cJson, cRequestId )

<i>    </i>LOCAL hReq        := {=>}
<i>    </i>LOCAL nDecoded    := 0
<i>    </i>LOCAL cResponse   := ""
<i>    </i>LOCAL cSource     := ""
<i>    </i>LOCAL cSender     := ""
<i>    </i>LOCAL cMessage    := ""
<i>    </i>LOCAL cReference  := ""
<i>    </i>LOCAL cStatus     := ""
<i>    </i>LOCAL hAuditRow   := {=>}
<i>    </i>
<i>    </i>// --- Body prüfen ---
<i>    </i>nDecoded := hb_jsonDecode( cJson, @hReq )
<i>    </i>IF ValType( nDecoded ) != "N" .OR. nDecoded <= 0
<i>    </i>    RETURN hb_jsonEncode({ "success" => .F., "error" => "Invalid Webhook JSON" })
<i>    </i>ENDIF

<i>    </i>// 1. Von wem kommt der Webhook? (MessageBird, Brevo oder Mollie)
<i>    </i>cSource := Upper( hb_HGetDef( hReq, "provider", "UNKNOWN" ) )

<i>    </i>// --- FALL A: EINE NACHRICHT (WhatsApp / SMS / Mail) ---
<i>    </i>IF cSource == "MESSAGEBIRD" .OR. cSource == "BREVO"
<i>    </i>    
<i>    </i>    // Die Nummer oder E-Mail des Gastes
<i>    </i>    cSender  := hb_HGetDef( hReq, "sender_address", "" ) 
<i>    </i>    // Der Text, den der Gast geschrieben hat
<i>    </i>    cMessage := hb_HGetDef( hReq, "message_text", "" )
<i>    </i>    
<i>    </i>    // OPTIONAL: Wenn wir direkt den Kunden finden wollen
<i>    </i>    // nKundenRecNo := FindCustomerByContact( cSender )
<i>    </i>    
<i>    </i>    // Neue Nachricht in die POSTEINGANG.DBF (oder AUDIT) speichern
<i>    </i>    USE ( "X:\xwhdaten\AUDIT\INBOX.DBF" ) NEW ALIAS INBOX SHARED
<i>    </i>    INBOX->( DbAppend() ) // Neue Zeile anhängen
<i>    </i>    
<i>    </i>    REPLACE INBOX->DATETIME WITH DToC(Date()) + " " + Time()
<i>    </i>    REPLACE INBOX->PROVIDER WITH cSource
<i>    </i>    REPLACE INBOX->SENDER   WITH cSender
<i>    </i>    REPLACE INBOX->MESSAGE  WITH Left(cMessage, 250) // Gekürzt auf DBF-Feldlänge
<i>    </i>    REPLACE INBOX->READ     WITH .F. // Ungelesen für die Rezeption!
<i>    </i>    
<i>    </i>    INBOX->( DbCommit() )
<i>    </i>    INBOX->( dbCloseArea() )
<i>    </i>    
<i>    </i>    // Antworte dem Provider sofort "OK", sonst schickt er das selbe Paket in 5 Min. nochmal!
<i>    </i>    RETURN hb_jsonEncode({ "success" => .T., "message" => "Inbox logged" })


<i>    </i>// --- FALL B: EINE ZAHLUNG (Mollie) ---
<i>    </i>ELSEIF cSource == "MOLLIE"
<i>    </i>    
<i>    </i>    // Mollie übergibt uns meistens eine eigene Referenz (z.B. deine Buchungsnummer)
<i>    </i>    cReference := hb_HGetDef( hReq, "booking_id", "" ) 
<i>    </i>    cStatus    := Upper( hb_HGetDef( hReq, "payment_status", "OPEN" ) )
<i>    </i>    
<i>    </i>    IF !Empty( cReference ) .AND. cStatus == "PAID"
<i>    </i>        // Buchung heraussuchen und als BEZAHLT markieren!
<i>    </i>        USE ( "X:\xwhdaten\DATAWIN\BELEGUNG.DBF" ) NEW ALIAS ZPLAN SHARED
<i>    </i>        SET ORDER TO TAG "BUCHUNGSNR"
<i>    </i>        
<i>    </i>        IF ZPLAN->( DbSeek( cReference ) )
<i>    </i>            IF ZPLAN->( RLock() )
<i>    </i>                REPLACE ZPLAN->FIXBUCHUNG WITH "BEZAHLT"
<i>    </i>                ZPLAN->( DbCommit() )
<i>    </i>                ZPLAN->( DbUnlock() )
<i>    </i>            ENDIF
<i>    </i>        ENDIF
<i>    </i>        ZPLAN->( dbCloseArea() )
<i>    </i>    ENDIF

<i>    </i>    RETURN hb_jsonEncode({ "success" => .T., "message" => "Payment processed" })

<i>    </i>ENDIF

<i>    </i>// Unbekannter Provider
<i>    </i>RETURN hb_jsonEncode({ "success" => .F., "error" => "Unknown Provider" })

Estimado Jonsson,

Se me ocurre que la solución ideal sería construir FWH64 como una DLL y asi poder usarla desde HIX.

De esta forma construimos FWH64 como una DLL. Tiene que ser de 64 bits ya que HIX es de 64 bits.

go64.bat

Code: Select all Collapse
@setlocal
call "%ProgramFiles%\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat" amd64
c:\harbour\bin\win\msvc64\hbmk2 fwh64.hbp -comp=msvc64
@endlocal

fwh64.hbp

Code: Select all Collapse
-gui
-hbdynvm

-ofwh64

-Ic:\fwh64\include

fwh64.prg

-Lc:\fwh64\lib

-lFiveH64
-lFiveHC64

-lgdiplus
-lole32
-lOleDlg
-lversion
-lucrt
-luxtheme

-cflag=-MD

xhb.hbc
hbct.hbc
hbwin.hbc
hbmzip.hbc
hbziparc.hbc
hbfoxpro.hbc

-ldflag+=/NODEFAULTLIB:libucrt
-ldflag+=/NODEFAULTLIB:LIBCMT

fwh64.prg

Code: Select all Collapse
function Main()

local o := TArrayData()

return nil

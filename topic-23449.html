Si, algo se preprocesa de una forma que a OutOfLuck no le gusta, sin embargo sus hermanos menores y otros clientes de correo no tienen problemas. Actualmente lo uso así:

Code (fw): Select all Collapse
              If cEsOutLook == 'NO'
                  DEFINE MAIL oMail ;
                                SUBJECT 'Copia de Seguridad de Contabilidad' ;
                                TEXT 'Generado por AtiConta 2.0 el '+DtoC(Date())+' a las '+Left( Time(), 5 ) ;
                                FILES ( cFichero ), ( cFichero ) ;
                                ORIGIN AllTrim( cMailUser ), AllTrim( cMailUser ) ;
                                TO 'Destino del correo', ( AllTrim( cCorreo ) )
               Else
                  DEFINE MAIL oMail ;
                                SUBJECT 'Copia de Seguridad de Contabilidad - '+cNombre ;
                                TEXT 'Generado por AtiConta 2.0 el '+DtoC(Date())+' a las '+Left( Time(), 5 ) ;
                                FILES ( cFichero ), ( cFichero ) ;
                                ORIGIN AllTrim( cMailUser ), AllTrim( cMailUser ) ;
                                TO ( AllTrim( cCorreo ) )


La sutileza es usar TO ( AllTrim( cCorreo ) ) o TO 'Destino del correo', ( AllTrim( cCorreo ) ), segun sea o no OutOfLuck

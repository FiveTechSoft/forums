Thanks for the one reply!

I am also sad.

I mean to say that I have been banging on about the appearance and operation of the FiveWin ListBox Browse for a while now (since FiveWin v2.2) and nothing has changed.

The ability to browse and display data contained in database files is fundamental to any dBase / Clipper / FiveWin / xHarbour application; it is in my opinion the core of any system we write; so why has this code not been looked at or changed since its inception in FiveWin 1.92 (Demo)?

I know there are other browses available from other users but I don't need the extra features and functionality that these provide (and I did not pay for them either).

The FiveWin ListBox Browse is ideal for most applications in my opinion and just needs to be tidied up a bit.

We have all come a long way since FiveWin 1.92 (Demo). I mean to say all of the other 'little' things have been fixed like Icons that sometime worked, sometimes not and sometimes dissapeared, and applications written with FiveWin and xHarbour do have that professional and polished look with the exception of the FiveWin ListBox Browse Class!!!

What is needed in my opinion is the following (and anyone else interested in this subject please feel free to add to this list):

1 - The ability to set the height of the rows and headings and the text of the headings to be spaced correctly in proportion to the chosen height and chosen font for the browse.

2 - The headings to appear the same as any other XP application that has a data browse (like MS Outlook for example) i.e. underlined when mouse is moved over them just like FiveWin Win32 Headers.

3 - Correct operation of the vertical scroll bar (been on about this for just as long).

4 - The option to choose whether or not you want lines displayed between each record or field (although this is not critical in my opinion).

5 - Correct alignment of the vertical lines between fields in relation to their headings (the vertical lines have always been slightly offset to the right since FiveWin 2.2 and does not look very nice).

Thats it!

How difficult can these changes be to make for the people who actually developed the libraries?

We already have the Win32 Headers so how much more effort can be involved in including these on a ListBox Browse?

Even if I knew how to make these changes myself what would be the point? Every time there is a change or update to WBrowse for example I would have to make my changes again so that any other new changes made to WBrowse would not be missed and then there is always the risk that my changes would interfere with changes made by FiveTech.

Please can I have some input on this subject?

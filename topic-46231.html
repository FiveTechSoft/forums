Tips para el poderoso editor Antigravity

Posted: Wed Dec 31, 2025 01:55 PM

Antigravity de Google es un Fork de VSCode de Microsoft con pequeños cambios para integrar el Agente IA de Geminis y otros.

Descargar e instalar https://antigravity.google/download

Aqui les dejo mi experiencia con instalaciones básicas

Code: Select all Collapse
INSTALAR LAS SIGUIENTES EXTENSIONES:
- Harbour and xHarbour de Antonino Perricone
- Project Manager Para cambiar de proyectos.
- VsCode Action Buttons Crea botones de acción en la barra de tareas
  Antigravity no la tiene. Descargar seunlanlege.action-buttons-1.2.3.vsix é instalar
  -En extensiones pulsar ... elegir Instalar desde VSIX
   -Ubicar la extensión descargada y dar Instalar

- CAMBIAR lenguaje Ctrl-Shift-P Configure Display Language Elegir el deseado
- Para que los caracteres especiales no sean cambiados por basura
  -Dar click en UTF-8 en la barra inferior
   -Volver a abrir en Encoding
    -Elegeir Windows 1252

-CARPETAS DE UN PROYECTO
 -Archivo -Abrir carpeta Ctrl+K+O (La principal)
 -Archivo -Agregar carpeta de trabajo (PRGs en otras carpetas)
 -Archivo -Guardar area de trabajo como (Ubicar donde guardar)
 -Ir a icono Project Manager y guardar el proyecto 
 
CONFIGURACIONES 
  Ir a -Archivo -Preferencias -Configuraciones
 - workbench.colorTheme, Elegir el color de tema de su agrado 
 - harbour.compilerExecutable, Colocar la ruta de HARBOUR.EXE
 - terminal.integrated.defaultProfile.windows, elegir Command prompt (CMD)
 - files.exclude, Para que no muestre archivos y carpetas
 - Auto Guess Encoding, Activar Para caracteres especiales.
 - Cursor blinking, elegir Solid ó el que desee
 - Breadcrumbs, Desactivar ruta del archivo abierto. 
 - Sticky Scroll, desactivar
 - Editor.occurrencesHighlight, En off para que no resalte la palabra en todo el archivo
 - Files.autoSave, Dar afterdelay para que guarde los cambios automáticamente
  

ATAJOS CON TECLAS
Ctrl+B Oculta el Explorador 
Ctrl+Alt+B Oculta el agente Geminis
Ctrl+Shift+K Borra la línea donde está el cursor
Ctrl+Z Deshacer cambios
Shift+Alt+A Poner un bloque seleccionado como comentario /*bloque*/
Ctrl+F Para hacer busqueda de texto
Ctrl+H Para Reemplazar Texto
Alt+flecha arriba o abajo Mueve la línea donde está el cursor
Ctrl+Shift+T Vuelve a abrir el último archivo cerrado
Ctrl+D Selecciona una palabra del texto

El archivo de configuración setting.json se guarda en: C:\Users\tusuario\AppData\Roaming\Antigravity\User

Les tiene que quedar parecido a esto

Code: Select all Collapse
{
    "workbench.colorTheme": "Visual Studio Light",
    "editor.minimap.enabled": false,
    "terminal.integrated.defaultProfile.windows": "Command Prompt",
    "harbour.compilerExecutable": "D:\\Sistemas\\Compiler\\HarbourVC\\Bin\\harbour",
    "includePath": [
        "${workspaceFolder}/**",
        "D:\\Sistemas\\Compiler\\FWH\\include"
    ],
    "files.exclude": {
        "folder_to_exclude": true,
        "OBJ": true,
        "**/*.i*": true,
        "**/*.e*": true,
        "**/*.c*": true,
        "**/*.h*": true,
        "**/*.r*": true,
        "**/*.o*": true,
        "**/*.prj": true,
        "**/*.z*": true,
        "**/*.code-workspace": true
    },
    "actionButtons": {
        "commands": [
            {
                "name": "♻️Compila",
                "color": "#ff0034",
                "command": "GoHbmk.bat ${fileBasenameNoExtension}",
                "cwd": "${workspaceFolder}"
            }
        ],
        "defaultColor": "white",
        "reloadButton": "↻",
        "loadNpmCommands": false
    },
    "files.autoGuessEncoding": true,
    "editor.cursorBlinking": "solid",

    "editor.occurrencesHighlight": "off",

    "security.workspace.trust.untrustedFiles": "open",
    "breadcrumbs.enabled": false,
    "editor.stickyScroll.enabled": false
}

El actionButtons lo deben modificar manualmente

Dear colleagues,

Is “line-by-line programming” — and a KI “co-worker” directly embedded in the IDE — today not rather a constraint?

Not because it is fundamentally wrong,
but because it may no longer represent the appropriate level of abstraction for modern application development.

Working line by line — even with an AI co-worker inside the IDE — often leads to:

a focus on syntax instead of intent

local correctness instead of system behavior

micro-decisions instead of architecture

loss of time on tasks that are not value-creating

This approach made sense when:

tools were still “dumb”

editors had no semantic understanding

little to no contextual information was available

These conditions have changed significantly.

_______________________________________________________________________________________

Estimados colegas,

¿No es hoy “programar línea por línea” — y un “co-worker” de IA integrado directamente en el IDE — más bien una limitación?

No porque sea incorrecto en sí,
sino porque quizá ya no representa el nivel de abstracción adecuado para el desarrollo moderno de aplicaciones.

Trabajar línea por línea — incluso con un co-worker de IA dentro del IDE — suele conducir a:

enfoque en la sintaxis en lugar de la intención

corrección local en lugar de comportamiento del sistema

microdecisiones en lugar de arquitectura

pérdida de tiempo en tareas que no aportan valor

Este enfoque tenía sentido cuando:

las herramientas eran todavía “simples”

los editores no tenían semántica

apenas había contexto disponible

Estas condiciones han cambiado de forma considerable.

Best regards,

Ribbon - No deshabilita boton
Posted: Thu Jun 12, 2014 08:53 PM
Gente
Uso una RibbonBar donde uno de sus Grupos tiene 3 botones.
De estos, según el usuario, se permite el acceso a todos o a 2 de ellos
El código del Ribbon es el siguiente
Code (fw): Select all Collapse
   // Pestaña CONTABLE (Cajas)
   ADD GROUP saoMn[_GrpCaja] RIBBON h0:oRBar TO OPTION 3 PROMPT "Cajas" WIDTH GetClientRect( oW0:hWnd )[4]-18
   @ 12, 10 ADD BUTTON saoMn[_BtCajaM] PROMPT "Define Caja" BITMAP "bmp_Master" TOP ROUND ;
            GROUP  saoMn[_GrpCaja] SIZE 80,60 ;
            ACTION oSelf:oCajaM:ShowCabCajas( oW0 )

   ADD SEPARATOR TO GROUP saoMn[_GrpCaja] COLUMN 100
   @ 12,110 ADD BUTTON PROMPT "Movimientos"+CRLF+"Maestro" BITMAP "bmp_MvtoCj" TOP ROUND ;
            GROUP  saoMn[_GrpCaja] SIZE 80,60 ;
            WHEN    h0:IDUser == _SUPER ;                                             // HABILITADO solo si es _SUPER
            ACTION oSelf:oCajaM:BrwMovimientos( oW0 )
   @ 12,220 ADD BUTTON PROMPT "Movimientos"+CRLF+"Trabajo" BITMAP "bmp_MvtoCj" TOP ROUND ;
            GROUP  saoMn[_GrpCaja] SIZE 80,60 ;
            ACTION oSelf:oCajaT:BrwMovimientos( oW0 )
   ...
Si a la App accedió _SUPER, entonces todo los botones estan habilitados

Luego, si cambia el Usuario hago lo siguiente:
Code (fw): Select all Collapse
   MENU oMenu POPUP
      MENUITEM  " Cerrar Sesión de '"+Rtrim( h0:User )+"'" ;
         RESOURCE "bmp_CerrarS" ;
         ACTION ( ::oPermisos:oUser:SetRes( "mgUsers2.dll" ), ;
                  if( !::oPermisos:oUser:ChkUser( , FALSE ) , ;
                      __Quit(), ;
                     ( h0:IDUser:= ::oPermisos:oUser:oDbUsr1:FUser01, ;
                       h0:User  := RTrim( ::oPermisos:oUser:oDbUsr1:FUser02 ), ;
                       h0:CjaAct:= ::oPermisos:oUser:oDbUsr1:FIDCaja01, ;
                       saoMn[_BtCajaM]:Refresh() ) ), ;                                              // Refresco el Botón en Cuestión
                  ::oPermisos:oUser:FreeRes() )
Ahora el btono, si paso el mouse por enciama se ve así:

No obstante al hacer clic sobre le boton el acceso esta deshabilitado. O sea, no obstante, se cumple el propósito y no se puede acceder
Lo extraño:
Si cambio de pestañas (tiene varias), no cambia el aspecto pero; si accedo desde cualquiera de los dos botones, abre sendos Dialgos, y luego cierro es Dialog; entoces SI el pintado del botón aparece como es deseado

Alex:

Tienes por lo menos 5 soluciones para lo que quieres hacer:

1) Si tu servidor tiene PHP, que creo recordar que si lo tiene, (es un Apache, ¿ correcto ?) puedes usar el cliente PHP de Advantage, hay cliente PHP para ADS desde Linux.

2) Si quieres utilizar Java y WebServices, desde el mismo Linux, puedes utilizar el cliente JDBC de Advantage, que viene con todo y codigo fuente, es un JDBC clase 2, pero funciona con productos como TomCat sin ningún problema, tengo un cliente que así lo usa y nunca se ha quejado.

3) Si quieres que todo corra en tu servidor Linux, puedes utilizar también CGIs con el mismo xHarbour para Linux con el RDD ADS y el ACE32 para Linux, funciona perfectamente.

4) Si quieres programar usando las herramientas que tienes en este momento (FW + xHarbour + ADS) puedes usar un servidor web "intermedio" basado en Windows (asi lo tenemos nosotros, nuestra web la tenemos en un servidor Apache y tenemos un servidor alterno donde están todos nuestros desarrollos web corriendo el Orelly WebSite)

5) Igual que lo anterior, puedes montar un servidor basado en Windows con Internet Information Server y usar el xHBScript con el cliente ASP de Advantage para accesar a tus datos en novell, solo tienes que asegurarte que tus equpos windows tengan el cliente Novell Instalado.

Claro, depende de tu version de ADS y de tu Linux. La que ustedes tienen es una 7.1 por lo que no tienes ningun problema en usar los clientes ADS correspondientes.

Hola Juan,

hay varias instancias en las que un correo puede clasificarse como Spam: El propio servidor de correos donde se recibe el mail, el cliente de correos que se usa para leer el mail, tu antivirus, etc.

Revisa las cabeceras del mensaje recibido, en ellas encontrarás las relacionadas con el Spam y te darán un a pista del porqué son clasificados como tal. Por ejemplo te pongo las cabeceras de un mensaje de mi cuenta:

Code (fw): Select all Collapse
X-Virus-Scanned: by amavisd-new-2.6.5 (20110407) (Debian) at atisa.es   -->quien está haciendo la verificación antispam, en mi caso el servidor que recibe el correo
X-Spam-Flag: NO          --> Resultado de la verificación
X-Spam-Score: 0.789   ---> puntaje obtenido segun las reglas de verificación de Spam
X-Spam-Level:
X-Spam-Status: No, score=0.789 required=4 tests=[BAYES_50=0.8,HTML_MESSAGE=0.001, SPF_HELO_PASS=-0.001, SPF_PASS=-0.001,T_RP_MATCHES_RCVD=-0.01] --> Las pruebas en las que el mail ha dado positivo y el peso relativo en puntaje. required=4 significa que el puntaje tiene que ser >= 4 para que sea considerado spam.


Tambien puede suceder que si mucha gente MANUALMENTE marca los mensajes de un dominio como spam, ese dominio cae en una BlackList que los servidores de correo consultan para poner filtros.
Acabo de verificar www.adaptaproyectoserp.com.ve y te tengo buenas noticias: ese dominio no está en la lista de spammers.

Code (fw): Select all Collapse
Blocklist Removal Center    
    
Blocklist Lookup Results

adaptaproyectoserp.com.ve is not listed in the DBL


Envíame un correo a carlos[punto]mora[arroba]atisa[punto]es desde ese PHP a ver si lo recibe com ospam y que dice la cabecera

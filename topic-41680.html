hbmk2 por defecto busca el compilador c que tengamos definidos en la variable path, por lo que lo mejor es efinir nuestros compiladores tanto c como harbour en un fichero bat que llame a nuestro hbmk

por ejemplo,aca descarto el path por defecto y lo reemplazo temporalmente por los que me interesa sean accesibles
durante la compulacion, luego lo restauro.


@echo off
set olpath=%path%
set path=%path%;d:\compiler\mingw\bin;d:\compiler\harbour\bin
hbmk2 dmolina3.hbp
set path=%olpath%

Esta llamada:

call "c:\Program Files\Microsoft Visual Studio\2019\Community\vc\Auxiliary\Build\vcvarsall.bat" x86

hbmk2 ... -comp=msvc

le estamos diciendo a hbmk2 que compilador tiene que usar

pero MSVC no puede estar en "cualquier lugar", tiene que estar donde MSVC se instala por defecto.

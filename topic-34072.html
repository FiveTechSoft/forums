Esempio di TXC.BAT ( batch di invio )

ECHO OFF
CLS

ECHO ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
ECHO :: TXC.BAT TRASMISSIONE ARTICOLI A DITRON ::
ECHO ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
ECHO :: PLUIC.TXT ::
ECHO ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
:START

ECHO .
ECHO ........ TRASMISSIONE INSERIMENTI IN CORSO ..........
ECHO .

ECHO ........ CASSA 1 ..........
ECHO .

CALL WEC INVIA.TXT /p1

ECHO .
ECHO ........ CASSA 2 ..........
ECHO .

CALL WEC INVIA.TXT /p2
:FINE

Contenuto di INVIA.TXT

; INVIA.TXT : Esempio di file-comando WinEcrCom
;
; Esempio che trasmette un file dati-articolo

;Trasmetto il file articoli
EM_LINK CMD='UA', FILE = 'C:\TX\PLUIC.TXT', ERR='C:\TX\ERRORI.TXT', BLOCK, OFFLINE

L'OCX gestito in FWH con ACTIVEX è più che stabile (ho parecchie installazioni e non mi danno problemi) ed è compatibile con il POS for .NET di Microsoft. Il Driver WINECRCOM come vedi nel .BAT ha una sua sintassi e comunque i due modi interagiscono con lo stesso SO (Service Object). Con WINECRCOM (in auto-run) devi prima configurare la barriera casse, indicare i nomi dei file comandi ed errori. Poi, lanciando il driver, la comunicazione con gli ECR è automatica(non devi fare altro che formarti il file con i comandi appropriati e copiarli nella cartella di interscambio
